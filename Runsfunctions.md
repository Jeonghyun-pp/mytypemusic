  3. 체크포인트 시스템

  파이프라인이 사람 입력/승인이 필요한 지점에서 멈추는 메커니즘.

  ┌───────────────────┬──────────┬──────────────────────┬──────────────────────────────────────────────────────┐
  │    Checkpoint     │   Type   │     UI 컴포넌트      │                         역할                         │
  ├───────────────────┼──────────┼──────────────────────┼──────────────────────────────────────────────────────┤
  │ slide_plan_review │ input    │ SlidePlanReview      │ LLM 생성 슬라이드 플랜 수정                          │
  ├───────────────────┼──────────┼──────────────────────┼──────────────────────────────────────────────────────┤
  │ design_editing    │ input    │ RunDesignEditor      │ 비주얼 디자인 편집 (Style/Code/AI/Ref + Preview)     │
  ├───────────────────┼──────────┼──────────────────────┼──────────────────────────────────────────────────────┤
  │ deck_review       │ approval │ DeckReviewPanel      │ 렌더링된 최종 덱 승인/거부                           │
  ├───────────────────┼──────────┼──────────────────────┼──────────────────────────────────────────────────────┤
  │ human_input       │ input    │ CheckpointPanel 본체 │ PostType별 크리에이티브 입력 (밈텍스트, 앨범선택 등) │
  ├───────────────────┼──────────┼──────────────────────┼──────────────────────────────────────────────────────┤

  └───────────────────┴──────────┴──────────────────────┴──────────────────────────────────────────────────────┘

  Checkpoint 상태: waiting → resolved / rejected

  5. Design 편집기 (RunDesignEditor)

  Run 내 design_editing 체크포인트에서 작동하는 비주얼 에디터:

  - 4개 탭: Style (프리셋/폰트무드/컬러), Code (HTML 직접편집), AI (Claude 채팅), Ref Images (레퍼런스 분석)
  - 실시간 프리뷰: Satori 기반 1080×1350 렌더링
  - Undo/Redo: 50단계, Ctrl+Z / Ctrl+Shift+Z
  - Auto-save: 1초 디바운스로 서버 저장
  - 데이터 브릿지: DeckEditSession ↔ DesignSpec 양방향 변환 (designBridge.ts)
  - 기존 Design 탭과 완전 독립: 같은 하위 컴포넌트를 import만 할 뿐, Design 탭 코드 수정 없음

  ---
  6. 웹 리서치 (web_research)

  webResearch.ts가 슬라이드별 키워드를 받아서:
  1. Naver 뉴스 + 블로그 + 백과 병렬 검색
  2. 슬라이드당 최대 5개 fact 추출 (제목, 설명, 출처, 링크, 날짜)
  3. web-research.json 저장 → LLM 프롬프트에 주입

  LLM이 구체적 수치·날짜·이름을 포함한 사실 기반 콘텐츠를 생성하게 됨.

  7. 데이터 흐름 요약

  [위저드]
    prompt + slideKeywords + heroFiles + referenceFiles
      ↓
  [web_research]  slideKeywords → Naver 검색 → web-research.json
      ↓
  [topic_request → intel_D]  prompt → topicId → topic.agent2.json
      ↓
  [content → bridge]  토픽분석 → validated-post.json
      ↓
  [llm_slide_plan]  topic + keywords + web facts → llm-slide-plan.json
      ↓
  [style_analysis]  reference-images → style-profile.json
      ↓
  [design_editing]  DeckEditSession ↔ DesignSpec (UI 편집)
      ↓
  [agent2_render]  HTML → PNG (1080×1350)
      ↓
  [deck_review]  최종 검토
      ↓
  [pre_publish_qa → bundle]  QA + 배포 번들
