## 1. 시작하기

### 1-1. 온보딩

처음 접속하면 온보딩 위저드가 나타납니다.

| 단계 | 내용 |
|------|------|
| Step 1 | SNS 계정 연결 (Threads, Instagram, X 등) |
| Step 2 | AI가 연결된 계정의 글쓰기 스타일을 자동 분석 (1~2분) |
| Step 3 | 분석된 페르소나 확인 후 콘텐츠 제작 시작 |

> 계정 연결 없이도 **"건너뛰고 바로 시작하기"** 버튼으로 디자인, 블로그, 캘린더 기능을 즉시 사용할 수 있습니다.

### 1-2. 화면 구성

- **좌측 사이드바**: 전체 메뉴 (Create, Plan, Publish, Manage, Analyze, Settings)
- **우측 하단 + 버튼**: 어디서든 통합 컴포저를 열어 빠르게 포스트 작성 & 발행
- **모바일**: 상단 햄버거 메뉴로 사이드바를 열 수 있습니다

## 2. 홈 대시보드

경로: **Home** (`/studio`)

대시보드는 "지금 해야 할 것 → 다음에 할 것 → 전체 그림" 순서로 구성됩니다.

### 오늘 할 일 (최상단)
- 미완료 일정 개수와 오늘 날짜 표시
- **파이프라인 상태 칩**: 대기 중 발행물, 승인 대기 제안, 미읽은 메시지 → 클릭하면 해당 페이지로 이동
- **오늘 일정 카드**: 오늘 예정된 콘텐츠 목록 (진행 중 > 예정 > 완료 순 정렬)

### AI 오늘의 제안
- 트렌드 + 브랜드 키워드 + 최근 인기 콘텐츠를 교차 분석하여 주제 3개를 추천
- 각 제안 카드에서:
  - **카드뉴스** → 디자인 에디터로 이동 (주제 + 카드뉴스 컨셉 자동 입력)
  - **블로그** → 블로그 에디터로 이동 (주제 + 아웃라인 자동 입력)
  - **빠른 발행** → 통합 컴포저가 열리며 SNS 텍스트 + 해시태그 자동 입력

### 이번 주 남은 일정
- 내일부터 주말까지의 콘텐츠 일정을 한눈에 확인

### 빠른 이동
- Design Editor, Reels Editor, Calendar, Database로 바로 이동

## 3. 콘텐츠 제작

### 3-1. Create Hub

경로: **Create Hub** (`/studio/create`)

주제 하나를 입력하면 3가지 포맷(SNS 포스트, 블로그 글, 카드뉴스)을 동시에 생성합니다.

1. 주제 입력 (예: "2026 여름 페스티벌 트렌드")
2. 페르소나 선택 (선택사항)
3. **"3가지 포맷 생성"** 클릭
4. 결과 카드에서 원하는 포맷을 선택하여 편집/발행으로 이동

### 3-2. Design Editor (카드뉴스)

경로: **Design** (`/studio/design`)

Satori 기반 카드뉴스 에디터입니다. 최대 10장의 슬라이드를 편집할 수 있습니다.

#### 기본 편집
| 기능 | 설명 |
|------|------|
| 슬라이드 관리 | 추가/삭제, 하단 네비게이션으로 이동 |
| 텍스트 편집 | 제목, 부제목, 본문, 푸터 입력 |
| 이미지 | 히어로 이미지 업로드, 표시 모드(cover/fill) 전환 |
| 스타일 | 텍스트 색상, 강조색, 배경 그라데이션, 그림자, 블러 |
| 폰트 | impact, editorial, clean-sans 등 무드 선택 |
| 프리셋 | news, beauty, tech, lifestyle, music 등 원클릭 스타일 적용 |
| 캔버스 크기 | 정사각형, 세로형 등 비율 선택 |

#### 고급 기능
| 기능 | 설명 |
|------|------|
| 레이어 시스템 | 텍스트/이미지/도형/SVG 레이어 추가, 순서 변경, 속성 편집 |
| 커스텀 HTML | 코드 모드에서 직접 HTML 입력 |
| DB 임포트 | 저장된 디자인 불러오기 |
| Undo/Redo | Ctrl+Z / Ctrl+Shift+Z (최대 50단계) |

#### 출력
| 기능 | 설명 |
|------|------|
| PNG 다운로드 | 전체 슬라이드를 PNG로 일괄 다운로드 |
| SNS 발행 | 슬라이드를 렌더링 후 QuickPublish 패널에서 바로 발행 |
| Figma 내보내기 | HTML을 클립보드에 복사 |

#### 프로젝트 저장
- 프로젝트 모드: DB에 자동 저장
- 일반 모드: localStorage에 자동 저장 (새로고침 후에도 유지)

### 3-3. Reels Editor (숏폼 영상)

경로: **Reels** (`/studio/reels`)

세로형(1080x1920) 숏폼 영상을 편집합니다.

1. 영상 파일을 드래그 앤 드롭 또는 파일 선택으로 업로드
2. **타임라인**에서 시작/종료 지점을 조정하여 트리밍
3. **텍스트 오버레이** 추가 (위치, 타이밍 설정)
4. 카테고리 선택
5. MP4 렌더링 (개발 중)

> 편집하는 동안 영상이 백그라운드에서 서버에 업로드됩니다.

### 3-4. Blog (AI 블로그 글)

경로: **Blog** (`/studio/blog`)

AI 에디토리얼 파이프라인을 통해 2000자 이상의 블로그 글을 자동 생성합니다.

**파이프라인 6단계:**
```
주제 입력 → 리서치(웹검색+KG) → 아웃라인 생성 → 페르소나 기반 작성 → 에디터 검수 → 완성
```

#### 사용법
1. 주제 입력 → **"블로그 글 생성"** 클릭 (1~2분 소요)
2. 생성된 글 목록에서 카드 클릭 → 에디터 진입
3. 에디터에서 편집:
   - 본문 내용 수정
   - SEO 제목 (60자 권장), 메타 설명 (160자 권장) 편집
   - SEO 키워드 확인
4. **저장** 또는 **SNS 발행**

### 3-5. Import (외부 링크 변환)

경로: **Import** (`/studio/import`)

웹 URL을 붙여넣으면 내용을 추출하여 SNS 포스트로 변환합니다.

1. URL을 줄 단위로 입력 (여러 개 가능)
2. 추가 지시사항 입력 (타겟, 톤, 스타일 등 - 선택)
3. **"콘텐츠 추출"** 클릭 → 각 URL의 제목, 도메인, 발췌문 확인
4. 플랫폼 선택 (Threads, Instagram, X, LinkedIn)
5. **"전체 포스트 생성"** 또는 개별 URL마다 포스트 생성
6. 결과를 클립보드에 복사하여 사용

## 4. 기획 & 리서치

### 4-1. Calendar (캘린더)

경로: **Calendar** (`/studio/calendar`)

월간 캘린더에서 콘텐츠 일정을 관리합니다.

#### 기본 사용
- 월 이동(←/→), 오늘 버튼으로 탐색
- 날짜의 이벤트 칩으로 일정 현황 한눈에 파악

#### 날짜 선택 시
- **빠른 제작 버튼** 3개가 나타남:
  - **빠른 포스트**: 통합 컴포저가 열리며, 예약 시간이 해당 날짜 오후 6시로 자동 세팅
  - **카드뉴스**: 디자인 에디터로 이동
  - **블로그**: 블로그 에디터로 이동
- 해당 날짜의 이벤트 목록 확인/편집/삭제
- 새 일정 추가 (제목, 타입, 카테고리, 상태, 메모)

### 4-2. Plan (AI 콘텐츠 기획)

경로: **Plan** (`/studio/plan`)

AI가 기간별 콘텐츠 발행 계획을 생성합니다.

#### 설정
| 항목 | 설명 |
|------|------|
| 기간 | 이번 주, 다음 주, 2주 등 프리셋 또는 직접 지정 |
| 주간 발행 횟수 | 총 포스트 수, 일일 최대 수 (1~5) |
| 집중 요일 | 발행이 많은 요일 지정 |
| 타입 비율 | Post, Reels, Promotion 비율 조정 (고급 설정) |
| 집중/제외 카테고리 | 원하는 주제에 가중치 부여 (고급 설정) |

#### 결과 활용
- 타임라인에서 생성된 일정 확인
- 개별 항목 편집 (날짜, 제목, 설명, 타입, 카테고리, 태그)
- **캘린더에 추가**: 개별 또는 일괄 추가
- **ICS 내보내기**: 외부 캘린더 앱에 가져오기

### 4-3. Research (레퍼런스 분석)

경로: **Research** (`/studio/research`)

SNS 게시물이나 디자인의 스크린샷을 업로드하면 AI가 스타일을 분석합니다.

1. **"+ 새 리서치"** 클릭
2. 소스 URL 또는 핸들 입력
3. 스크린샷 업로드 (드래그 앤 드롭, 최대 10장) 또는 텍스트 입력
4. **"분석"** 클릭 → AI 분석 리포트 생성
5. 리포트 목록에서 과거 분석 결과 열람/삭제

### 4-4. Database (디자인 레퍼런스)

경로: **Database** (`/studio/database`)

디자인 레퍼런스를 수집하고 관리하는 라이브러리입니다.

| 탭 | 기능 |
|----|------|
| **References** | 카테고리별 디자인 레퍼런스 업로드/관리. 이미지 + 제목 + 설명 |
| **Mood Search** | 무드/분위기 키워드로 비주얼 레퍼런스 검색 |
| **Spotify** | 음악/아티스트 검색으로 앨범 커버 등 뮤직 레퍼런스 탐색 |

## 5. 발행

### 5-1. 통합 컴포저 (+ 버튼)

화면 우하단의 **초록색 + 버튼**을 클릭하면 어디서든 열 수 있습니다.

#### 텍스트 탭
- 캡션 입력 + 해시태그 (쉼표/공백 구분)
- 글자 수 실시간 표시

#### 미디어 탭
- 이미지 URL 추가 (썸네일 미리보기)
- 카드뉴스 만들기 / 릴스 만들기 바로가기

#### 채널 선택
- 연결된 SNS 계정에서 발행할 채널 체크박스 선택 (기본: 전체 선택)

#### 발행 모드
| 모드 | 설명 |
|------|------|
| **지금 발행** | 선택한 모든 채널에 즉시 발행 |
| **예약** | 날짜/시간 지정. **AI 추천 시간** 버튼으로 참여율 기반 최적 시간 자동 세팅 |
| **드래프트** | 임시 저장 (나중에 Publish 페이지에서 관리) |

### 5-2. Publish (발행 관리)

경로: **Publish** (`/studio/publish`)

#### 새 발행물 작성
1. SNS 계정 선택
2. 발행 내용 입력 + 해시태그
3. **즉시 발행** 또는 **예약 발행** (AI 추천 시간 활용 가능)

#### 추천 발행 시간
- 계정 선택 시 과거 참여율 데이터 기반 **최적 시간대** 자동 표시
- 요일 + 시간 + 참여율(%) 표시
- 클릭하면 예약 시간에 자동 반영

#### 발행 내역
- 상태별 필터: 전체, 대기, 예약, 발행완료, 실패
- 각 발행물의 상태, 플랫폼, 내용 미리보기, 시간 확인
- 발행 완료 시 원본 게시물 링크 제공
- 대기/예약/실패 상태의 발행물 삭제 가능

### 5-3. Autopilot (자동 발행)

경로: **Autopilot** (`/studio/autopilot`)

AI가 자동으로 콘텐츠를 생성하고 발행합니다.

#### 설정 만들기
1. SNS 계정 선택
2. 승인 모드: **수동 승인** (추천) 또는 자동 발행
3. 일일 발행 수 (1~5)
4. 주제 키워드 입력 (쉼표 구분)
5. 플랫폼 지정

#### 콘텐츠 제안 관리
- AI가 생성한 콘텐츠 제안 카드 목록
- 각 제안에서: 주제, AI 추론 이유, 본문 미리보기
- **승인** → 예약 발행 / **거절** → 폐기

## 6. 관리

### 6-1. Inbox (메시지 수신함)

경로: **Inbox** (`/studio/inbox`)

연결된 SNS 계정의 댓글과 DM을 한 곳에서 관리합니다.

#### 메시지 탭
- **필터**: 전체, 미읽음, 질문, 칭찬, 불만, 스팸
- 각 메시지에서:
  - 발신자, 플랫폼, 분류 배지, 감정 색상, 우선순위 표시
  - **답장**: 직접 작성 또는 **AI 추천 답변** 사용
  - 읽음 처리

#### 자동 답장 규칙 탭
| 설정 항목 | 설명 |
|-----------|------|
| 계정 선택 | 어떤 계정에 적용할지 |
| 트리거 유형 | 분류(질문/불만 등), 키워드, 감정, 전체 |
| AI 응답 | AI가 자동으로 답변을 생성할지 여부 |
| 답변 템플릿 | AI를 사용하지 않을 경우 고정 답변 |

### 6-2. Campaigns (캠페인)

경로: **Campaigns** (`/studio/campaigns`)

키워드 기반 자동 댓글 캠페인을 관리합니다.

1. 계정 선택 + 캠페인 이름
2. 키워드 입력 (쉼표 구분)
3. 일일 한도 설정 (5~30)
4. AI 지시사항 입력 (톤, 스타일 등)
5. 이용약관 확인 후 시작/중지
6. 댓글 로그에서 실행 결과 확인 (성공/실패/대기)

> 플랫폼 이용약관을 준수하여 적절한 일일 한도를 설정하세요.

## 7. 분석

### 7-1. Analytics (성과 분석)

경로: **Analytics** (`/studio/analytics`)

#### 대시보드 탭
- **요약 카드**: 총 팔로워(증감), 총 도달, 총 인게이지먼트, 평균 참여율, 발행 수
- **계정별 상세**: 팔로워/도달/참여율 차트 (선형 + 막대)
- **인구통계**: 성별, 연령대, 상위 도시, 상위 국가
- **인기 주제**: 성과 좋은 주제 + 이유 + 해당 주제로 콘텐츠 제작 바로가기

#### AI 인사이트 탭
- 챗봇 인터페이스로 SNS 성과에 대해 질문
- 예시 질문:
  - "지난주 참여율 변화는?"

## 8. 설정

### 8-1. Persona (글쓰기 페르소나)

경로: **Persona** (`/studio/persona`)

AI가 콘텐츠를 작성할 때 사용하는 글쓰기 스타일을 정의합니다.

| 생성 방법 | 설명 |
|-----------|------|
| **수동 입력** | 스타일 설명을 직접 작성 |
| **템플릿** | 4종 프리셋 중 선택 (프로 마케터, 캐주얼 크리에이터, 교육 전문가, 오피니언 리더) |
| **텍스트 분석** | 기존 글 2개 이상을 붙여넣으면 AI가 스타일을 자동 추출 |

#### 페르소나 속성
- **톤**: 격식, 유머, 감정, 에너지 수치
- **어휘/구조**: 자주 쓰는 표현, 문장 구조 패턴
- **스타일 지문**: 전체적인 글쓰기 성격 요약

#### 테스트
- 페르소나를 선택하고 주제를 입력하면 해당 페르소나 스타일로 샘플 텍스트 생성

### 8-2. Accounts (SNS 계정)

경로: **Accounts** (`/studio/accounts`)

#### 연결 가능한 플랫폼
| 사용 가능 | 준비 중 |
|-----------|---------|
| Threads, Instagram, X, YouTube, TikTok, LinkedIn, WordPress | Facebook, Pinterest, Telegram |

#### 계정 관리
- 연결된 계정: 프로필 이미지, 플랫폼, 표시 이름, 토큰 만료 상태
- **연결 해제** 버튼으로 계정 제거
- 플랫폼 버튼 클릭 → OAuth 인증 → 자동 연결

## 핵심 워크플로우

### A. 빠른 SNS 포스트 (1분)
```
+ 버튼 → 텍스트 입력 → 채널 선택 → 지금 발행
```

### B. AI 제안으로 카드뉴스 발행 (5분)
```
홈 > AI 제안 > 카드뉴스 클릭 → 디자인 에디터에서 편집 → SNS 발행 버튼 → 채널 선택 → 발행
```

### C. 블로그 자동 생성 (5분)
```
Blog > 주제 입력 > 블로그 글 생성 (1~2분 대기) → 에디터에서 편집 → 저장/발행
```

### D. 주간 콘텐츠 기획 (10분)
```
Plan > 기간/빈도 설정 > 생성 → 타임라인 확인 → 캘린더에 일괄 추가
```

### E. 캘린더에서 콘텐츠 제작 (3분)
```
Calendar > 빈 날짜 클릭 > 빠른 포스트 → 통합 컴포저 (예약 시간 자동 세팅) → 예약 발행
```

### F. 외부 콘텐츠 재활용 (3분)
```
Import > URL 붙여넣기 > 추출 → 플랫폼 선택 > 포스트 생성 → 복사하여 사용
```

### G. 완전 자동화
```
Accounts에서 계정 연결 → Persona 생성 → Autopilot 설정(키워드+일일 발행 수) → AI가 자동 생성 → 승인만 하면 발행
```
