# 인스타그램 참고 계정 기반 콘텐츠 제안 고도화 계획

## 배경

현재 Autopilot 스캐너(`src/lib/autopilot/scanner.ts`)가 생성하는 콘텐츠 제안이 너무 포괄적이다.

**현재 문제 예시:**
- "인디 음악의 숨은 명곡 발굴하기" → 어떤 곡?
- "K팝 아티스트의 비하인드 스토리 공개!" → 누구?
- "2026년 봄, 기대되는 신보 소식!" → 어떤 아티스트의 어떤 앨범?

**목표:**
- "잔나비 '주저하는 연인들을 위해' 라이브 vs 스튜디오 비교 분석"
- "뉴진스 3월 컴백 트레일러 분석 — 'How Sweet' 뮤비 레퍼런스 추적"

처럼 **구체적인 아티스트명, 곡명, 날짜, 이벤트**가 포함된 제안이 필요하다.

---

## 개선 방향 3가지

### 1. 프롬프트에 구체적 데이터 주입

DB에 이미 존재하는 데이터를 활용한다.

| 데이터 소스 | 모델 | 활용 |
|------------|------|------|
| 아티스트/앨범 정보 | `MusicArtist`, `MusicAlbum` | 최근 등록된 아티스트/앨범을 프롬프트에 포함 |
| 주제 성과 | `TopicPerformance` | 이미 다룬 주제 제외, 성과 좋은 카테고리 우선 |
| 트렌드 스냅샷 | `TrendSnapshot` | 기존과 동일 |

### 2. 최근 제안 이력 중복 방지

- `AutopilotProposal`에서 최근 7일간 생성된 제안 목록을 프롬프트에 포함
- "아래 주제는 이미 다뤘으니 제외하라"고 명시

### 3. 참고 인스타 계정 피드 분석

다른 음악 매거진/크리에이터 계정의 최근 게시물을 수집하여 프롬프트에 참고 자료로 제공한다.

## 상세 구현 계획

### Phase 1: 프롬프트 개선 (토큰 불필요, 즉시 가능)

**파일:** `apps/studio/src/lib/autopilot/scanner.ts`

#### 1-1. DB 데이터 조회 추가

```typescript
// 최근 등록/업데이트된 아티스트 (최대 10명)
const recentArtists = await prisma.musicArtist.findMany({
  orderBy: { updatedAt: "desc" },
  take: 10,
  select: { name: true, nameKo: true, genres: true },
});

// 최근 앨범 (최대 10개)
const recentAlbums = await prisma.musicAlbum.findMany({
  orderBy: { createdAt: "desc" },
  take: 10,
  include: { artist: { select: { name: true, nameKo: true } } },
});

// 최근 7일 제안 이력
const recentProposals = await prisma.autopilotProposal.findMany({
  where: {
    autopilotConfigId: configId,
    createdAt: { gte: new Date(Date.now() - 7 * 24 * 60 * 60 * 1000) },
  },
  select: { topic: true },
});

// 이미 다룬 주제 (성과 기록)
const coveredTopics = await prisma.topicPerformance.findMany({
  where: { lastPublishedAt: { not: null } },
  orderBy: { lastPublishedAt: "desc" },
  take: 20,
  select: { topic: true, category: true, avgEngagement: true },
});
```

#### 1-2. 프롬프트 구체성 제약 조건 추가

```
규칙:
1. 반드시 특정 아티스트명, 곡명, 앨범명, 또는 이벤트 날짜를 포함할 것
2. "~의 매력", "~알아보기", "~발굴하기" 같은 포괄적 제목 금지
3. 아래 "최근 다룬 주제"와 겹치지 않을 것
4. 아래 "참고할 아티스트/앨범 데이터"를 적극 활용할 것
```

### Phase 2: 참고 계정 모델 + UI (토큰 불필요)

#### 2-1. DB 모델 추가

**파일:** `apps/studio/prisma/schema.prisma`

```prisma
model ReferenceAccount {
  id        String   @id @default(uuid())
  platform  String   // "instagram" | "youtube" | "x"
  username  String   // "@indiepost_kr" 등
  category  String   @default("inspiration") // "competitor" | "inspiration" | "industry"
  note      String   @default("") // 이 계정을 참고하는 이유
  isActive  Boolean  @default(true)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  feeds     ReferenceFeed[]

  @@unique([platform, username])
  @@map("reference_accounts")
}

model ReferenceFeed {
  id                 String   @id @default(uuid())
  referenceAccountId String
  account            ReferenceAccount @relation(fields: [referenceAccountId], references: [id], onDelete: Cascade)
  platformPostId     String   // 플랫폼별 게시물 ID
  caption            String   @db.Text
  likeCount          Int      @default(0)
  commentCount       Int      @default(0)
  postedAt           DateTime
  fetchedAt          DateTime @default(now())

  @@unique([referenceAccountId, platformPostId])
  @@index([referenceAccountId])
  @@index([postedAt])
  @@map("reference_feeds")
}
```

#### 2-2. API 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/reference-accounts` | 참고 계정 목록 |
| POST | `/api/reference-accounts` | 참고 계정 추가 |
| DELETE | `/api/reference-accounts/[id]` | 참고 계정 삭제 |
| POST | `/api/reference-accounts/[id]/sync` | 수동 피드 동기화 |

#### 2-3. Studio UI

Autopilot 설정 페이지에 "참고 계정" 섹션 추가:
- 계정 추가/삭제
- 카테고리(경쟁사/영감/업계) 선택
- 최근 수집된 게시물 미리보기
- 수동 동기화 버튼

### Phase 3: Instagram Business Discovery API 연동 (토큰 필요)

#### 3-0. 토큰 발급 절차

1. **Instagram 계정 전환**: 설정 > 계정 > 프로페셔널 계정으로 전환 (비즈니스 또는 크리에이터)
2. **Facebook 페이지 연결**: Instagram 설정 > 계정 센터 > Facebook 페이지 연결
3. **Meta Developer 앱 생성**: https://developers.facebook.com > 앱 만들기 > 비즈니스 유형
4. **권한 추가**:
   - `instagram_basic` — 자신의 프로필/미디어 접근
   - `instagram_manage_insights` — Business Discovery 사용에 필요
   - `pages_read_engagement` — 페이지 토큰 기반 조회
5. **장기 토큰 발급**:
   - Graph API Explorer에서 단기 토큰 생성
   - `/oauth/access_token?grant_type=fb_exchange_token` 으로 60일 장기 토큰 교환
   - 앱에서 자동 갱신 로직 구현 (기존 `SnsAccount.refreshToken` 활용)
6. **SnsAccount에 등록**: Studio UI에서 Instagram 계정 연결 또는 수동으로 DB에 토큰 저장

#### 3-3. scanner.ts 통합

```typescript
// 참고 계정 인기 게시물 (최근 7일, 반응 높은 순)
const referencePosts = await prisma.referenceFeed.findMany({
  where: { postedAt: { gte: sevenDaysAgo } },
  orderBy: { likeCount: "desc" },
  take: 15,
  include: { account: { select: { username: true, category: true } } },
});

// 프롬프트에 추가
const referenceContext = referencePosts
  .map(p => `[@${p.account.username}] ${p.caption.slice(0, 150)} (좋아요 ${p.likeCount})`)
  .join("\n");
```

프롬프트에 포함 시 제약:
```
참고 계정의 최근 인기 게시물:
${referenceContext}

규칙:
- 위 게시물의 주제를 그대로 복사하지 말 것
- 위 게시물에서 영감을 받되, 우리만의 독자적 각도로 재해석할 것
- 위 게시물이 다루는 아티스트/이벤트는 시의성이 검증된 것이므로 적극 참고
```
