# Studio 사용 가이드

Web Magazine Studio는 AI 기반 콘텐츠 기획, 제작, 발행, 분석을 하나의 워크플로우로 통합한 웹 애플리케이션입니다.

---

이 가이드는 두 가지 방식으로 사용할 수 있습니다:

- **가이드 파이프라인** — 처음부터 끝까지 순서대로 따라가는 단계별 가이드
- **기능별 레퍼런스** — 각 기능을 독립적으로 참조하는 사전식 가이드

---

## 가이드 파이프라인: 첫 콘텐츠 발행까지

처음 사용하는 경우, 아래 8단계를 순서대로 따라가면 첫 콘텐츠를 발행할 수 있습니다.
각 단계는 이전 단계의 결과물을 다음 단계의 입력으로 사용합니다.

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  STEP 1         STEP 2         STEP 3         STEP 4                │
│  초기 설정  ──→  페르소나  ──→  키워드   ──→  주제 발견              │
│  (계정 연동)     (글쓰기 톤)    (니치 등록)    (대시보드 AI 제안)     │
│                                                                      │
│  STEP 5         STEP 6         STEP 7         STEP 8                │
│  기획    ──→    제작     ──→   발행     ──→   분석 & 개선            │
│  (Plan+Cal)     (Create+Design) (Publish)     (Analytics)            │
│                                                                      │
│              ↑                                         │              │
│              └─────────── 반복 (매주/매일) ─────────────┘              │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### STEP 1. 초기 설정 — 계정 연동 (최초 1회)

> 소요 시간: 5~10분
> 경로: Settings > Accounts (`/studio/accounts`)

**이 단계의 목적**: 콘텐츠를 발행할 SNS 플랫폼을 연결합니다.

1. Studio에 로그인합니다
2. 첫 접속 시 **온보딩 위자드**가 자동으로 나타납니다
3. "계정 연결" 버튼 클릭 → Accounts 페이지로 이동
4. 원하는 플랫폼의 "연결" 클릭 → OAuth 인증 → 완료
5. 최소 1개 이상 연결해야 발행 기능이 동작합니다

**필요한 사전 준비**: 각 플랫폼의 API 키가 환경변수에 설정되어 있어야 합니다.
→ [API Keys Guide](./api-keys-guide.md) 참조

**결과물**: 연결된 SNS 계정 (Publish, Autopilot, Inbox에서 사용)

---

### STEP 2. 페르소나 설정 — 글쓰기 톤 정의 (최초 1회)

> 소요 시간: 3~5분
> 경로: Settings > Persona (`/studio/persona`)

**이 단계의 목적**: AI가 글을 생성할 때 사용할 톤과 스타일을 정의합니다.

**가장 빠른 방법 — 템플릿 선택:**
1. "템플릿" 탭 클릭
2. 4가지 중 하나 선택 (예: Casual Creator)
3. 저장

**가장 정확한 방법 — 기존 글 분석:**
1. "분석" 탭 클릭
2. 자신이 과거에 쓴 글을 붙여넣기
3. AI가 자동으로 스타일 핑거프린트 생성
4. "테스트" 탭에서 주제를 입력해 샘플 확인
5. 마음에 들면 저장

**결과물**: 기본 페르소나 → Create Hub, Autopilot에서 자동 적용

---

### STEP 3. 니치 키워드 등록 (최초 1회, 이후 수시 업데이트)

> 소요 시간: 1분
> 경로: 홈 대시보드 (`/studio`) 하단 AI 제안 영역

**이 단계의 목적**: AI가 내 전문 분야에 맞는 주제를 추천하도록 합니다.

1. 홈 대시보드의 AI 제안 영역에서 키워드 편집 클릭
2. 전문 분야 키워드 입력 (예: `인디밴드`, `K-pop`, `페스티벌`)
3. 저장

**결과물**: 니치 키워드 → 대시보드 AI 제안이 내 분야에 맞춤화

---

### STEP 4. 주제 발견 — AI 트렌드 제안 확인

> 소요 시간: 2~3분
> 경로: 홈 대시보드 (`/studio`)

**이 단계의 목적**: 오늘 어떤 콘텐츠를 만들지 결정합니다.

1. 대시보드의 **AI 콘텐츠 제안** 영역 확인
2. **일반 제안** 3개 + **니치 제안** 3개가 표시됨
3. 각 제안에는 SNS/블로그/카드뉴스 미리보기 포함
4. 마음에 드는 제안을 클릭하면:
   - "카드뉴스" → Design Editor로 바로 이동 (주제+슬라이드 내용 자동 전달)
   - "블로그" → Blog Editor로 이동 (주제+아웃라인 자동 전달)
   - "포스트" → Publish로 이동 (텍스트 자동 전달)
   - "빠른 발행" → Unified Composer 팝업 (즉시 발행)

**결과물**: 선택한 주제 → 다음 단계의 에디터에 자동 입력

> 대안: AI 제안 대신 직접 주제를 정했다면, Create Hub(`/studio/create`)로 바로 이동

---

### STEP 5. 기획 — 주간 계획 수립 (선택)

> 소요 시간: 5분
> 경로: Plan Generator (`/studio/plan`) → Calendar (`/studio/calendar`)

**이 단계의 목적**: 이번 주에 몇 개의 콘텐츠를 어떤 날짜에 발행할지 계획합니다.
단발성 콘텐츠라면 이 단계를 건너뛰고 STEP 6으로 갑니다.

1. **Plan Generator** 에서:
   - 기간 설정 (예: 이번 주 월~금)
   - 빈도 설정 (예: 하루 1개)
   - 카테고리/타입 비율 설정
   - "생성" 클릭 → AI가 날짜별 콘텐츠 계획 생성
2. 각 항목의 **"캘린더에 추가"** 클릭
3. **Calendar** 에서 등록된 일정 확인 및 조정

**결과물**: 캘린더에 등록된 주간 콘텐츠 일정

---

### STEP 6. 제작 — 콘텐츠 만들기

> 소요 시간: 10~30분 (콘텐츠 타입에 따라)
> 경로: 포맷에 따라 다름

콘텐츠 타입에 따라 아래 경로 중 하나를 선택합니다:

#### 경로 A: SNS 포스트 (가장 빠름)

```
Create Hub → 주제 입력 → SNS 포스트 자동 생성 → Quick Publish로 즉시 발행
```

1. Create Hub (`/studio/create`)에서 주제 입력
2. 페르소나 선택 (STEP 2에서 설정한 것)
3. "생성" 클릭 → SNS 텍스트+해시태그 자동 생성
4. Quick Publish 패널에서 계정 선택 → 발행 (STEP 7로 이동할 필요 없음)

#### 경로 B: 카드뉴스 (가장 일반적)

```
Create Hub → 카드뉴스 컨셉 생성 → Design Editor → 디자인 편집 → 내보내기/발행
```

1. Create Hub에서 주제 입력 → 카드뉴스 컨셉 생성
2. "카드뉴스" 링크 클릭 → **Design Editor**로 이동 (주제+슬라이드 자동 전달)
3. Design Editor에서:
   - **Style 탭**: 프리셋/폰트/색상 조정
   - **References 탭**: AI 이미지 생성 → "히어로 이미지로 사용" 클릭
   - **AI Chat 탭**: "배경 더 어둡게" 같은 자연어 수정
   - 슬라이드 추가/복제/삭제로 장수 조절
4. 완성 후:
   - "다운로드" → PNG/PDF 저장
   - "SNS 발행" 버튼 → 에디터 내 발행 패널에서 바로 발행 (STEP 7로 이동할 필요 없음)

#### 경로 C: 블로그 글

```
Create Hub → 블로그 아웃라인 생성 → Blog Editor → SEO 설정 → 저장
```

1. Create Hub에서 주제 입력 → 블로그 아웃라인 생성
2. "블로그" 링크 클릭 → **Blog Editor**로 이동 (주제+아웃라인 자동 전달)
3. Blog Editor에서:
   - AI 생성 초안 확인 및 수정
   - SEO 필드 설정 (메타 제목, 설명, 키워드)
   - 저장

#### 경로 D: 숏폼 영상

```
Reels Editor → 영상 업로드 → 텍스트 오버레이 → 트림 → 내보내기
```

1. Reels Editor (`/studio/reels`)에서 영상 업로드
2. 텍스트 오버레이 추가 (위치, 타이밍 설정)
3. 트림으로 길이 조절
4. 내보내기

#### 경로 E: 외부 기사 큐레이션

```
Import → URL 입력 → 추출 → SNS 포스트 생성 → 발행
```

1. Import (`/studio/import`)에서 기사 URL 붙여넣기
2. "추출" → "생성" 클릭
3. 생성된 포스트 확인 후 발행

**결과물**: 완성된 콘텐츠 (카드뉴스 PNG, 블로그 글, SNS 텍스트 등)

---

### STEP 7. 발행 — SNS에 올리기

> 소요 시간: 1~2분
> 경로: Publish (`/studio/publish`) 또는 에디터 내 Quick Publish

STEP 6에서 에디터 내 Quick Publish를 사용했다면 이 단계는 이미 완료된 상태입니다.

별도로 발행하는 경우:

1. Publish (`/studio/publish`) 이동
2. 계정 선택 (복수 플랫폼 동시 발행 가능)
3. 텍스트, 해시태그 입력 (STEP 6에서 넘어온 경우 자동 입력됨)
4. 미디어 URL 추가 (카드뉴스 PNG 등)
5. 발행 모드 선택:
   - **지금 발행**: 즉시 발행
   - **예약**: 날짜/시간 지정 → "스마트 타임" 클릭하면 AI 추천 시간 자동 적용
   - **임시저장**: 나중에 발행

**결과물**: 발행된 콘텐츠 (발행 이력에서 상태 확인 가능)

---

### STEP 8. 분석 & 개선 — 성과 확인

> 소요 시간: 5분
> 경로: Analytics (`/studio/analytics`)

**이 단계의 목적**: 발행한 콘텐츠의 성과를 확인하고 다음 콘텐츠에 반영합니다.

1. **Dashboard 탭**에서 핵심 지표 확인:
   - 팔로워 증감, 도달 수, 참여율
   - 인기 토픽 순위
2. **Chat 탭**에서 AI에게 분석 요청:
   - "이번 주 가장 반응이 좋았던 포스트는?"
   - "어떤 시간대에 올려야 참여율이 높아?"
3. 인사이트를 다음 주 기획(STEP 5)에 반영

**→ STEP 4로 돌아가서 반복**

---

### 반복 사이클 요약

| 빈도 | 단계 | 내용 |
|------|------|------|
| **최초 1회** | STEP 1~3 | 계정 연동, 페르소나, 키워드 |
| **매주 1회** | STEP 5 → 8 | 주간 기획 → 분석 리뷰 |
| **매일** | STEP 4 → 6 → 7 | 주제 발견 → 제작 → 발행 |

---

### 자동화 파이프라인 (Autopilot)

위 수동 파이프라인을 매일 반복하기 어렵다면, **Autopilot**으로 STEP 4~7을 자동화할 수 있습니다:

```
[1회 설정]
Settings > Accounts (계정 연동)
  → Settings > Persona (페르소나 설정)
  → Autopilot 설정 (계정, 키워드, 일일 횟수, 승인 모드)

[매일 자동]
AI가 트렌드 기반 주제 발견 (= STEP 4)
  → AI가 콘텐츠 생성 (= STEP 6)
  → 제안 카드로 표시 (수동 승인 모드)
  → 승인 시 자동 발행 (= STEP 7)

[주 1회 확인]
대시보드에서 제안 승인/거절
  → Analytics에서 성과 확인 (= STEP 8)
  → 키워드/페르소나 조정
```

Autopilot 설정 방법은 [5.2 Autopilot](#52-autopilot-자동-발행) 참조.

## 1. 초기 설정 (최초 1회)

Studio의 모든 기능을 사용하려면 아래 설정을 먼저 완료해야 합니다.
설정 순서대로 진행하세요.

### 1.1 로그인

Studio에 처음 접속하면 비밀번호 입력 화면이 나타납니다.
관리자가 설정한 사이트 비밀번호를 입력하면 30일간 로그인이 유지됩니다.

> 비밀번호는 환경변수 `SITE_PASSWORD`로 설정합니다.
> 미설정 시 비밀번호 없이 접근 가능합니다.

#### 화면 구조

- **좌측 사이드바**: 6개 그룹(CREATE, PLAN, PUBLISH, MANAGE, ANALYZE, SETTINGS)으로 구분된 메뉴
- **메인 영역**: 선택한 페이지의 콘텐츠
- **모바일**: 768px 이하에서 사이드바가 드로어로 변환

---

### 1.2 SNS 계정 연동 (SETTINGS > Accounts)

경로: `/studio/accounts`

> **왜 먼저?** — 계정이 연결되어야 발행(Publish), 자동발행(Autopilot), 메시지 관리(Inbox), 분석(Analytics) 기능이 동작합니다.

#### 사전 준비

각 플랫폼의 개발자 콘솔에서 OAuth 앱을 먼저 만들고 API 키를 환경변수에 설정해야 합니다.
자세한 키 발급 방법은 [API Keys Guide](./api-keys-guide.md)를 참고하세요.

| 환경변수 | 플랫폼 |
|----------|--------|
| `META_APP_ID`, `META_APP_SECRET` | Threads, Instagram |
| `X_CLIENT_ID`, `X_CLIENT_SECRET` | X (Twitter) |
| `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` | YouTube |
| `LINKEDIN_CLIENT_ID`, `LINKEDIN_CLIENT_SECRET` | LinkedIn |
| `TIKTOK_CLIENT_KEY`, `TIKTOK_CLIENT_SECRET` | TikTok |
| `WORDPRESS_COM_CLIENT_ID`, `WORDPRESS_COM_CLIENT_SECRET` | WordPress |

배포 환경에서는 `OAUTH_CALLBACK_BASE_URL`을 실제 도메인으로 변경해야 합니다:
```
OAUTH_CALLBACK_BASE_URL=https://mytypemusic.vercel.app
```

#### 연결 방법

1. 원하는 플랫폼의 **"연결"** 버튼 클릭

### 1.3 글쓰기 페르소나 설정 (SETTINGS > Persona)

경로: `/studio/persona`

> **왜 먼저?** — 페르소나를 설정해야 Create Hub, Autopilot 등에서 AI가 일관된 톤으로 글을 생성합니다. 미설정 시 기본 톤으로 동작하지만, 브랜드 정체성을 위해 먼저 설정하는 것을 권장합니다.

#### 생성 방법 4가지

**1. 템플릿 선택 (가장 빠름)**

| 템플릿 | 특징 |
|--------|------|
| Professional Marketer | 전문적이고 신뢰감 있는 마케팅 톤 |
| Casual Creator | 친근하고 자연스러운 크리에이터 톤 |
| Educational Expert | 정보 전달에 초점을 맞춘 교육적 톤 |
| Provocative Opinion Leader | 도발적이고 강한 의견을 가진 톤 |

**2. 글 분석 (가장 정확함)**
- 기존에 자신이 쓴 글 샘플을 붙여넣기
- AI가 글쓰기 스타일을 자동 분석
- 톤 속성과 스타일 핑거프린트 자동 생성

**3. 수동 생성**
- 이름 입력
- 스타일 핑거프린트 직접 작성 (예: "간결하고 위트 있는 음악 저널리스트 톤")

**4. 테스트**
- 생성된 페르소나에 주제를 입력하면 샘플 글 생성
- 원하는 스타일이 맞는지 확인 후 조정

#### 페르소나 관리
- 여러 개 생성 가능
- 기본 페르소나 설정 (default) — AI 생성 시 자동 적용
- 활성/비활성 토글

### 1.4 니치 키워드 등록 (홈 대시보드)

## 2. 홈 대시보드

경로: `/studio`

Studio에 접속하면 가장 먼저 보이는 화면입니다. 오늘 할 일과 AI 추천을 한눈에 확인합니다.

### 구성 요소

| 영역 | 설명 |
|------|------|

- 각 제안에는 SNS 포스트, 블로그 아웃라인, 카드뉴스 컨셉 미리보기 포함

### 새 글 작성 (FAB 버튼)

화면 우하단의 **"+"** 버튼을 클릭하면 **Unified Composer**(통합 작성기)가 열립니다.
어떤 페이지에서든 빠르게 SNS 포스트를 작성하고 발행할 수 있습니다.

### 3.1 Plan Generator (AI 콘텐츠 기획)

경로: `/studio/plan`

AI가 지정한 기간의 콘텐츠 발행 계획을 자동 생성합니다.

#### 사용법

1. **기간 설정**: 시작일 ~ 종료일
2. **빈도 설정**: 일별/주별 발행 횟수
3. **콘텐츠 선호도 설정**:
   - 집중 카테고리 선택 (복수 선택 가능)
   - 회피 카테고리 선택
   - 타입 비율 조절 (포스트/릴스/프로모션 %)
   - 추가 메모
4. **"생성"** 클릭
5. **결과 확인**: 날짜별 콘텐츠 타임라인
   - 각 항목: 날짜, 제목, 타입, 카테고리, 설명
   - **"캘린더에 추가"** 버튼으로 개별 항목을 캘린더에 등록

### 3.2 Calendar (콘텐츠 캘린더)

경로: `/studio/calendar`

- **사이드 패널**: 선택한 날짜의 이벤트 목록 + 생성/편집 폼

#### 이벤트 관리

1. **날짜 클릭**: 해당 날짜 선택 → 사이드 패널에 이벤트 표시
2. **이벤트 생성**:
   - "빠른 포스트" 또는 "예약 포스트" 버튼 클릭
   - 제목, 타입(포스트/릴스/프로모션), 카테고리, 메모 입력
   - 저장
3. **이벤트 편집**: 이벤트 클릭 → 수정 폼
4. **이벤트 삭제**: 이벤트의 삭제 버튼

#### 이벤트 타입 & 상태

| 타입 | 배지 색상 |
|------|-----------|
| post | 파란색 |
| reels | 보라색 |
| promotion | 주황색 |

| 상태 | 의미 |
|------|------|
| planned | 기획됨 |

| published | 발행 완료 |

## 4. 만들기 — CREATE

기획이 끝나면 실제 콘텐츠를 제작합니다.

### 4.1 Create Hub (통합 생성)

경로: `/studio/create`

하나의 주제로 SNS 포스트, 블로그 아웃라인, 카드뉴스 컨셉을 **동시에** 생성합니다.
무엇을 만들지 정했다면 여기서 시작하세요.

#### 사용법

1. **주제 입력**: 콘텐츠 주제를 입력 (예: "2026 봄 인디 페스티벌 트렌드")
2. **페르소나 선택**: 드롭다운에서 글쓰기 스타일 선택 (선택사항, [1.3](#13-글쓰기-페르소나-설정-settings--persona)에서 설정)
3. **생성 버튼** 클릭
4. 3가지 포맷이 동시에 생성됩니다:
   - **SNS 포스트**: 본문 + 해시태그 (바로 발행 가능)
   - **블로그 아웃라인**: 제목 + 핵심 포인트 구조
   - **카드뉴스 컨셉**: 제목 + 슬라이드 구성 + 장수

#### 빠른 발행

생성된 SNS 포스트 아래의 **Quick Publish** 패널에서:
1. 발행할 계정 선택 (체크박스)
2. "발행" 버튼 클릭
3. 성공/실패 결과 확인

> 사전 준비: SNS 계정 연동 필요 ([1.2](#12-sns-계정-연동-settings--accounts))

#### 상세 편집으로 이동

각 포맷 카드의 링크를 클릭하면 해당 전용 에디터로 이동:
- SNS → Publish 페이지
- 블로그 → Blog Editor
- 카드뉴스 → Design Editor

### 4.2 Design Editor (카드뉴스 디자인)

경로: `/studio/design`

카드뉴스, 인포그래픽, SNS 이미지를 디자인하는 전문 에디터입니다.

#### 진입 방식

| 모드 | 설명 |
|------|------|
| **프로젝트 브라우저** | 저장된 디자인 프로젝트 목록 (그리드) |
| **Quick Editor** | 주제 입력 → AI가 즉시 카드뉴스 생성 |
| **프로젝트 편집** | 기존 프로젝트 클릭하여 편집 |

#### 에디터 화면 구성

```
┌─────────────────────────────────────────────────────────┐
│  헤더: 카테고리 | 캔버스 크기 | 참조 | 내보내기/다운로드  │
├────────────────────────────────┬────────────────────────┤
│                                │  우측 패널 (380px)      │
│                                │  ┌──────────────────┐  │
│      슬라이드 미리보기 (캔버스)    │  │ Style | AI Chat  │  │
│                                │  │ References       │  │
│                                │  │ Code | Layers    │  │
│                                │  │ Database         │  │
│                                │  │ DataViz          │  │
│                                │  └──────────────────┘  │
├────────────────────────────────┴────────────────────────┤
│  슬라이드 네비게이션: [1] [2] [3] [+추가] [복제] [삭제]    │
└─────────────────────────────────────────────────────────┘
```

#### 우측 패널 탭 상세

**Style 탭** — 디자인 스타일 전체 조정
- 프리셋 선택 (미리 정의된 디자인 스타일)
- 폰트 무드 선택 (bold-display, elegant-serif 등)
- 글로벌 스타일 조정:
  - 배경 그라데이션
  - 텍스트/강조/푸터 색상
  - 제목/본문 크기 (px), 굵기
  - 자간, 스크림 투명도, 이미지 밝기, 카드 라운딩

**AI Chat 탭** — 채팅으로 디자인 수정
- 자연어로 디자인 수정 요청 (예: "배경을 어둡게 바꿔줘", "제목 크기 키워줘")
- AI가 스타일, 텍스트, 레이아웃을 자동 수정

**References 탭** — 참고 이미지 & AI 이미지 생성
- 참고 이미지 업로드 (최대 3장, 드래그 앤 드롭)
- AI 이미지 생성:
  - 프롬프트 입력
  - 프로바이더 선택 (DALL-E / Flux)
  - 비율 선택 (가로/정사각/세로)
  - 생성된 이미지를 **"히어로 이미지로 사용"** 버튼으로 슬라이드 배경에 적용

> 사전 준비: DALL-E는 `OPENAI_API_KEY`, Flux는 `FAL_KEY` 환경변수 필요

**Code 탭** — HTML 직접 편집
- 슬라이드의 HTML 코드를 직접 편집 (Monaco Editor)
- JSX style 문법 지원 (`style={{ }}` → 자동 변환)

**Layers 탭** — 레이어 기반 편집
- 텍스트, 이미지, 도형 레이어 추가/편집
- 레이어 순서 변경, 위치/크기 조절

**Database 탭** — 자료실에서 가져오기
- [Database](#34-database-디자인-자료실)에 저장된 참조 이미지를 배경/요소로 가져오기

**DataViz 탭** — 인포그래픽 템플릿
- 차트, 비교표, 타임라인 등 데이터 시각화 템플릿

#### 캔버스 크기

| 프리셋 | 크기 | 용도 |
|--------|------|------|
| Instagram 피드 | 1080x1350 | 기본값 |
| Instagram 스토리 | 1080x1920 | 세로 풀사이즈 |
| 정사각형 | 1080x1080 | 범용 |
| 커스텀 | 자유 입력 | |

#### 내보내기

- **PNG 다운로드**: 각 슬라이드를 개별 PNG로 저장
- **전체 내보내기**: 모든 슬라이드를 ZIP으로 다운로드
- **PDF 내보내기**: 슬라이드를 PDF 문서로 저장

#### 키보드 단축키

| 단축키 | 동작 |
|--------|------|
| `Ctrl+Z` | 실행 취소 |
| `Ctrl+Shift+Z` | 다시 실행 |

### 4.3 Blog Editor (블로그)

경로: `/studio/blog`

SEO 최적화된 블로그 글을 작성합니다.

#### 목록 화면

- 저장된 블로그 글 카드 그리드
- 각 카드에 제목, 상태 (초안/발행), 단어 수 표시

#### 편집 화면

1. **주제 입력 후 AI 생성**: 주제만 입력하면 AI가 전체 글 초안 생성
2. **직접 편집**: 텍스트 에디터에서 본문 수정
3. **SEO 필드 설정**:
   - 메타 제목 (검색 결과에 표시될 제목)
   - 메타 설명 (검색 결과 요약문)
   - 키워드 (쉼표로 구분)
4. **상태 변경**: 초안 ↔ 발행
5. 자동 저장 지원

---

### 4.4 Reels Editor (숏폼 영상)

경로: `/studio/reels`

짧은 영상을 편집하고 텍스트 오버레이를 추가합니다.

#### 사용법

1. **영상 업로드**: 파일 선택 또는 드래그 앤 드롭
2. **카테고리 선택**: 콘텐츠 분류 선택
3. **텍스트 오버레이 추가**:
   - "텍스트 추가" 버튼 클릭
   - 텍스트 내용 입력
   - 위치 (x, y), 크기, 색상, 폰트 설정
   - 시작/종료 시간 설정 (타임라인)
4. **트림(자르기)**: 슬라이더로 시작/종료 지점 조절
5. **미리보기**: 플레이어에서 실시간 확인
6. **내보내기**: 최종 영상 다운로드

---

### 4.5 Import (URL 일괄 가져오기)

경로: `/studio/import`

외부 기사/블로그 URL에서 콘텐츠를 추출하고 SNS 포스트를 자동 생성합니다.

#### 사용법

1. URL을 입력 (쉼표 또는 줄바꿈으로 구분, 여러 개 가능)
2. (선택) AI 생성 지시사항 입력 (예: "음악 관점에서 요약해줘")
3. **"추출"** 버튼 클릭 → 각 URL에서 제목, 요약, 도메인 추출
4. **"생성"** 버튼 클릭 → URL별로 SNS 포스트 자동 생성
5. 결과 확인: 성공/실패 표시

---

## 5. 발행하기 — PUBLISH

콘텐츠가 완성되면 SNS에 발행합니다.

> 사전 준비: SNS 계정 연동 필요 ([1.2](#12-sns-계정-연동-settings--accounts))

### 5.1 Publish (SNS 발행)

경로: `/studio/publish`

다중 플랫폼에 동시에 콘텐츠를 발행합니다.

#### 새 포스트 작성

1. **계정 선택**: 발행할 SNS 계정 체크 (복수 선택 가능)
2. **본문 작성**: 텍스트 입력
3. **해시태그 입력**: 해시태그 추가
4. **미디어 추가**: 이미지/영상 URL 입력
5. **발행 모드 선택**:

| 모드 | 설명 |
|------|------|
| 지금 발행 | 즉시 발행 |
| 예약 | 날짜/시간 지정 후 예약 발행 |
| 임시저장 | 발행하지 않고 저장만 |

6. **스마트 타임 추천**: AI가 참여율 기반 최적 발행 시간 추천 → 클릭하면 자동 적용

#### 발행 이력

- 상태별 필터: 전체 / 임시저장 / 예약 / 발행됨 / 실패
- 각 항목에 표시:
  - 플랫폼 아이콘
  - 상태 배지
  - 발행 시간
  - 에러 메시지 (실패 시)
  - 발행된 포스트 링크 (성공 시)

---

### 5.2 Autopilot (자동 발행)

경로: `/studio/autopilot`

AI가 자동으로 콘텐츠를 제안하고, 승인 후 발행합니다.
매일 수동으로 글을 쓰지 않아도 일정한 발행 빈도를 유지할 수 있습니다.

> 사전 준비:
> - SNS 계정 연동 ([1.2](#12-sns-계정-연동-settings--accounts))
> - 페르소나 설정 권장 ([1.3](#13-글쓰기-페르소나-설정-settings--persona))

#### 설정 (Config)

1. **새 오토파일럿 생성**:
   - 계정 선택
   - 발행 플랫폼 선택 (복수)
   - 일일 발행 수
   - 승인 모드:
     - **수동**: 제안을 검토 후 직접 승인
     - **자동**: 생성 즉시 자동 발행
   - 주제 키워드 입력 (AI가 이 키워드 기반으로 주제 생성)
2. 설정 수정/삭제

#### 제안 보드 (Proposals)

- AI가 생성한 콘텐츠 제안이 카드로 표시
- 각 카드:
  - 주제 & 근거
  - 생성된 텍스트
  - 예약 시간
  - 상태 (대기중/승인/거절/발행됨)
- 액션:
  - **승인**: 발행 대기열에 추가
  - **거절**: 제안 폐기
  - **바로 발행**: 즉시 발행

---

## 6. 관리하기 — MANAGE

발행 후 들어오는 반응과 커뮤니티를 관리합니다.

> 사전 준비: SNS 계정 연동 필요 ([1.2](#12-sns-계정-연동-settings--accounts))

### 6.1 Inbox (메시지 관리)

경로: `/studio/inbox`

SNS 플랫폼에서 받은 메시지를 통합 관리합니다.

#### Messages 탭

- **메시지 목록**: 모든 플랫폼의 메시지를 한 곳에서 확인
- **필터**: 읽지 않음, 분류별 (질문/피드백/불만/스팸)
- 각 메시지에 표시:
  - 발신자 이름/핸들, 플랫폼
  - AI 자동 분류 (질문, 피드백, 불만, 스팸)
  - 감정 분석 (긍정/부정/중립)
  - 우선순위 (낮음/보통/높음/긴급)
- **답장**: 메시지 선택 → 답장 입력 → 전송
- **읽음 처리**: 읽음/안읽음 토글

#### Rules 탭 (자동 답장 규칙)

자동 답장 규칙을 설정합니다:

1. **규칙 생성**:
   - 트리거 타입: 분류 기반 (예: 질문이면) 또는 키워드 기반
   - 트리거 값: 특정 분류명 또는 키워드
   - 답장 템플릿: 고정 텍스트
   - AI 사용 토글: 켜면 상황에 맞는 답장을 AI가 동적 생성
   - 활성화/비활성화
2. 규칙 편집/삭제

---

### 6.2 Campaigns (자동 댓글 캠페인)

경로: `/studio/campaigns`

키워드 기반으로 관련 게시물에 자동 댓글을 달아 참여도를 높입니다.

#### 캠페인 생성

1. 계정 선택
2. 캠페인 이름 입력
3. 키워드 입력 (쉼표로 구분, 이 키워드가 포함된 게시물 대상)
4. 일일 댓글 제한 수
5. 운영 시간대 설정 (시작/종료 시간)
6. AI 지시사항 (댓글 톤/스타일 가이드)

#### 캠페인 관리

- 활성/비활성 토글
- 오늘의 댓글 수 / 일일 제한 표시
- 상세 보기: 활동 로그, 대상 게시물 URL, 상태

---

## 7. 분석하기 — ANALYZE

경로: `/studio/analytics`

발행된 콘텐츠의 성과를 분석합니다.

> 사전 준비: SNS 계정 연동 필요 ([1.2](#12-sns-계정-연동-settings--accounts)). 발행된 콘텐츠가 있어야 의미 있는 데이터가 표시됩니다.

#### Dashboard 탭

| 지표 | 설명 |
|------|------|
| 최신 팔로워 수 | 현재 팔로워 |
| 팔로워 증감 | 기간 대비 변화량 |
| 총 도달 수 | 콘텐츠가 노출된 횟수 |
| 참여율 | 좋아요+댓글+공유 / 도달 |

- **타임라인 차트**: 팔로워, 도달, 참여율의 시계열 그래프
- **인구통계**: 연령대, 성별, 지역 분포
- **인기 토픽**: 참여율이 높았던 주제 순위

#### Chat 탭

AI에게 자연어로 분석을 요청합니다:
- "이번 주 가장 반응이 좋았던 포스트는?"
- "팔로워 증가율이 떨어진 이유를 분석해줘"
- "다음 주 최적의 발행 시간을 추천해줘"

## 8. 공통 기능: Unified Composer (통합 작성기)

어디에서든 **"+" FAB 버튼** 또는 **"빠른 포스트"** 버튼으로 열 수 있는 팝업 작성기입니다.

### 기능

1. **계정 선택**: 열릴 때 활성 계정 모두 자동 선택
2. **텍스트 편집**: 본문 작성
3. **해시태그**: 해시태그 입력
4. **미디어**: 이미지/영상 URL 추가 (탭 전환)
5. **발행 모드**:
   - 지금 발행
   - 예약 발행 (날짜/시간 선택)
   - 임시저장
6. **스마트 타임**: AI 추천 발행 시간 자동 적용
7. **결과 확인**: 성공/실패 건수 토스트 알림

## 9. 워크플로우 예시

### 워크플로우 1: 처음 시작하기

```
1. 로그인
2. Settings > Accounts — SNS 계정 연동
3. Settings > Persona — 글쓰기 스타일 설정
4. 홈 대시보드 — 니치 키워드 등록
5. 홈 대시보드 — AI 제안 확인하며 첫 콘텐츠 제작
```

### 워크플로우 2: 주간 콘텐츠 기획 → 제작 → 발행

```
Plan Generator (1주일 계획 생성)
  → Calendar에 등록
  → 매일 캘린더 확인하며 Create Hub에서 콘텐츠 제작
  → Design Editor에서 카드뉴스 디자인
  → Publish에서 발행
  → Analytics에서 성과 분석
```

### 워크플로우 3: 트렌드 기반 카드뉴스 (빠른 제작)

```
홈 대시보드 (AI 제안 확인)
  → Create Hub (주제로 카드뉴스 컨셉 생성)
  → Design Editor (슬라이드 디자인)
  → References 탭에서 AI 이미지 생성 → 히어로로 적용
  → PNG 다운로드 또는 Publish에서 발행
```

### 워크플로우 4: 블로그 + SNS 동시 발행

```
Create Hub (주제 입력 → 블로그 + SNS 동시 생성)
  → Blog Editor (SEO 필드 채우고 본문 수정)
  → Quick Publish 패널로 SNS 즉시 발행
```

### 워크플로우 5: 자동화된 일일 콘텐츠

```
Persona 설정 (글쓰기 스타일 정의)
  → Autopilot 설정 (계정, 키워드, 일일 횟수)
  → 매일 대시보드에서 AI 제안 확인 → 승인/거절
  → 승인된 콘텐츠 자동 발행
  → Inbox에서 반응 관리
```

### 워크플로우 6: 외부 기사 큐레이션

```
Import (관심 기사 URL 일괄 입력)
  → AI가 SNS 포스트 자동 생성
  → Quick Publish로 즉시 발행
```
